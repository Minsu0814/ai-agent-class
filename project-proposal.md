# UAM 최적 버티포트 입지 분석 및 멀티모달 경로 탐색 시스템

## 1. 프로젝트 개요

현재 서울시 대중교통 데이터(스마트카드, GTFS)를 분석하여 **대중교통으로 불편한 구간**을 식별하고, **UAM(도심항공교통) 도입 시 통행 효율 개선 효과**를 분석하는 AI 기반 웹 서비스를 개발한다.

### 핵심 질문
> "서울시에서 UAM이 가장 필요한 구간은 어디이며, 도입 시 얼마나 효과적인가?"

---

## 2. 프로젝트 구조

```
스마트카드 데이터          GTFS 데이터
   (수요/OD)              (노선/네트워크)
       ↓                       ↓
  ┌─────────────────────────────────┐
  │  현재 대중교통 통행 분석         │
  │  - 장거리 but 오래 걸리는 OD    │
  │  - 환승 3회 이상 구간           │
  │  - 첫차/막차 취약 구간          │
  └──────────────┬──────────────────┘
                 ↓
  ┌─────────────────────────────────┐
  │  UAM 도입 후보 구간 도출        │
  │  - 직선거리 vs 대중교통 시간    │
  │  - 수요(스마트카드) 기반 우선순위│
  │  - 버티포트 최적 위치 제안      │
  └──────────────┬──────────────────┘
                 ↓
  ┌─────────────────────────────────┐
  │  LLM 기반 분석 인터페이스       │
  │  "강남↔인천공항 UAM 도입하면    │
  │   시간이 얼마나 줄어?"          │
  └─────────────────────────────────┘
```

---

## 3. 주요 기능

| 기능 | 사용 데이터 | 설명 |
|------|------------|------|
| 대중교통 불편 구간 히트맵 | 스마트카드 + GTFS | 통행시간 대비 직선거리 비율이 높은 OD 쌍 시각화 |
| 버티포트 입지 추천 | 스마트카드 (수요) | 클러스터링 기반 수요 밀집 지점 → 후보지 도출 |
| UAM 도입 전/후 비교 | GTFS + UAM 가상노선 | 특정 OD의 통행시간 단축 효과 시뮬레이션 |
| AI 질의응답 | LLM API | 자연어로 구간 분석 및 효과 예측 질의 |

## 4. 데이터

| 데이터 | 출처 | 용도 |
|--------|------|------|
| 서울시 스마트카드 데이터 | [교통카드 빅데이터 통합정보시스템](https://stcis.go.kr) | OD 수요 분석, 통행 패턴 파악 |
| 대중교통 GTFS 데이터 | [국가교통DB](https://www.ktdb.go.kr) | 노선 네트워크 분석, 경로 탐색 |
| 서울시 교통사고 데이터 | [TAAS](https://taas.koroad.or.kr) | 안전성 분석 (보조) |
| 서울시 유동인구 데이터 | [서울열린데이터광장](https://data.seoul.go.kr) | 수요 검증 |

## 5. 기술 스택

```
Frontend : Streamlit + Folium / Pydeck (지도 시각화)
Backend  : Python + Pandas + NetworkX (경로 분석)
AI       : Claude or GPT API (질의응답 + 데이터 분석)
Data     : 스마트카드 CSV + GTFS + UAM 가상노선
```

---

## 6. 기대 효과

- **시의성**: 서울시 K-UAM 실증 사업과 연계 가능한 분석 결과 제공
- **데이터 기반 의사결정**: 감이 아닌 실제 통행 데이터 기반 버티포트 입지 제안
- **Agentic AI 활용**: LLM이 분석 도구를 직접 호출하여 자연어 질의에 응답
- **시각적 결과물**: 지도 기반 대시보드로 직관적 데모 가능
