# AI Agent 기반 UAM 버티포트 입지 분석 및 도입 효과 평가 시스템

## 1. 프로젝트 개요

### 1.1 프로젝트 주제

서울시 대중교통 데이터(스마트카드, GTFS)를 활용하여 **UAM(도심항공교통) 버티포트 최적 입지를 분석**하고, **UAM 도입 전/후 교통 접근성 변화를 평가**하는 AI Agent 기반 분석 시스템을 개발한다.

### 1.2 핵심 질문

> "서울시에서 UAM이 가장 필요한 구간은 어디이며, 도입 시 교통 접근성과 형평성에 어떤 변화가 있는가?"

### 1.3 왜 이 주제인가

| 문제 인식 | 설명 |
|-----------|------|
| 기존 연구 한계 | 설문/인구밀도 기반 수요 추정 → 실제 통행 데이터 미반영 |
| UAM 단독 분석 | 기존 대중교통 네트워크와의 연계 분석 부족 |
| 분석의 단편성 | 시간 절감만 분석 → 접근성·형평성 등 다차원 평가 부재 |
| 분석 접근성 | 전문가만 가능한 분석 → 비전문가도 자연어로 분석 가능한 시스템 필요 |

## 3. 모델, Tool, 알고리즘

### 3.1 분석 알고리즘

| 단계 | 알고리즘 / 모델 | 용도 |
|------|-----------------|------|
| OD 분석 | Pandas 기반 집계 | 스마트카드 → OD 행렬 구축 |
| 네트워크 구축 | NetworkX + GTFS 파싱 | 대중교통 그래프 모델링 |
| UAM 수요 추정 | Logit Model (이산선택모델) | 대중교통 vs UAM 수단분담 |
| 버티포트 후보지 | K-Means / DBSCAN | 수요 밀집 지점 클러스터링 |
| 입지 최적화 | MILP (PuLP/Gurobi) | 접근성 최대화 + 형평성 제약 |
| 접근성 측정 | 등시선(Isochrone) 분석 | 일정 시간 내 도달 가능 범위 |
| 형평성 분석 | Gini 계수, Theil Index | 접근성 분포 균등도 측정 |
| 전/후 비교 | 시나리오 시뮬레이션 | 버티포트 수/요금/간격별 효과 |

### 3.2 기술 스택

```

웹 프레임워크 : Streamlit
지도 시각화   : Folium / Pydeck

최적화       : PuLP (MILP)
통계/ML     : Scikit-learn, Statsmodels
```

## 4. LLM Agent 설계

### 4.1 Agent 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                    사용자 (자연어 질의)                    │
│  "강남역에서 인천공항까지 UAM 타면 얼마나 빨라져?"         │
└──────────────────────┬──────────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────────┐
│                  Orchestrator Agent                       │
│  Role: 사용자 질의를 분석하고 적절한 Sub-Agent에 위임      │
│  - 질의 의도 파악 (분석 / 시각화 / 비교 / 추천)           │
│  - 실행 계획 수립 (어떤 도구를 어떤 순서로 호출할지)       │
│  - 결과 종합 및 사용자 응답 생성                          │
└────┬─────────┬──────────┬──────────┬─────────────────────┘
     ↓         ↓          ↓          ↓
┌────────┐┌─────────┐┌──────────┐┌──────────┐
│ OD     ││Network  ││Optimizer ││Evaluator │
│ Agent  ││ Agent   ││  Agent   ││  Agent   │
└────────┘└─────────┘└──────────┘└──────────┘
```

### 4.2 Agent별 Role & Task

#### Orchestrator Agent (총괄)
```
Role  : 사용자 질의를 해석하고, Sub-Agent들을 조율하여 최종 답변 생성
Task  :
  - 자연어 질의 → 의도 분류 (OD분석 / 경로탐색 / 최적화 / 효과평가)
  - 필요한 Sub-Agent 선택 및 실행 순서 결정
  - Sub-Agent 결과를 종합하여 자연어 + 시각화로 응답
Model : Claude API (또는 GPT-4)
```

#### OD Analysis Agent (통행 분석)
```
Role  : 스마트카드 데이터를 분석하여 OD 패턴, 수요 정보 제공
Task  :
  - 특정 지역/시간대 OD 수요 조회
  - 장거리·고시간 통행 필터링 (UAM 전환 후보)
  - 통행 패턴 통계 산출
Tools : [query_od_matrix, filter_long_trips, get_demand_stats]
```

#### Network Agent (네트워크 분석)
```
Role  : GTFS 기반 대중교통 네트워크에서 경로 탐색 및 접근성 분석
Task  :
  - 두 지점 간 대중교통 최단 경로/시간 계산
  - 등시선(Isochrone) 분석: 특정 지점에서 N분 내 도달 가능 범위
  - UAM 링크 추가 시 네트워크 재구성
Tools : [find_shortest_path, calc_isochrone, add_uam_links, calc_accessibility]
```

#### Optimizer Agent (최적화)
```
Role  : 버티포트 최적 입지를 결정
Task  :
  - 클러스터링으로 후보지 선정
  - MILP 모델로 최적 버티포트 배치 결정
  - 목적함수: 총 접근성 최대화 (+ 형평성 제약)
Tools : [cluster_candidates, optimize_vertiport, get_optimization_result]
```

#### Evaluator Agent (효과 평가)
```
Role  : UAM 도입 전/후 효과를 비교 평가
Task  :
  - 도입 전 접근성 측정 (대중교통만)
  - 도입 후 접근성 측정 (대중교통 + UAM)
  - 전/후 비교: 통행시간, 접근성, 형평성 변화
  - 시나리오별 민감도 분석
Tools : [measure_before, measure_after, compare_scenarios, calc_equity]
```

### 4.3 Agent 호출 흐름 예시

```
사용자: "강남역에서 인천공항까지 UAM 도입 효과를 분석해줘"

Orchestrator:
  1. 의도 파악 → "특정 OD 구간 UAM 효과 평가"
  2. 실행 계획:
     ① OD Agent → 강남-인천공항 현재 통행 데이터 조회
     ② Network Agent → 현재 대중교통 최단 경로/시간 계산
     ③ Network Agent → UAM 링크 추가 후 통합 경로/시간 계산
     ④ Evaluator Agent → 전/후 비교 (시간 절감, 접근성 변화)
  3. 결과 종합 → 자연어 응답 + 지도 시각화

응답:
  "강남역 → 인천공항 현재 대중교통 소요시간: 약 85분
   UAM 도입 시 (강남 버티포트 → 인천공항 버티포트): 약 35분
   시간 절감: 50분 (59% 단축)
   [지도: 두 경로 비교 시각화]"
```

### 4.4 Tool 목록

| Tool 이름 | 입력 | 출력 | 담당 Agent |
|-----------|------|------|-----------|
| `query_od_matrix` | 출발지, 도착지, 시간대 | OD 수요량, 통행시간 분포 | OD Agent |
| `filter_long_trips` | 거리 임계값, 시간 임계값 | UAM 전환 후보 OD 리스트 | OD Agent |
| `get_demand_stats` | 지역, 기간 | 수요 통계 (평균, 피크 등) | OD Agent |
| `find_shortest_path` | 출발지, 도착지, 모드 | 최단 경로, 소요시간, 환승 수 | Network Agent |
| `calc_isochrone` | 중심점, 시간 제한 | 도달 가능 범위 (GeoJSON) | Network Agent |
| `add_uam_links` | 버티포트 위치 리스트 | 통합 네트워크 | Network Agent |
| `calc_accessibility` | 네트워크, 기회 데이터 | 접근성 지표 (지역별) | Network Agent |
| `cluster_candidates` | OD 데이터, K | 버티포트 후보지 좌표 | Optimizer Agent |
| `optimize_vertiport` | 후보지, 수요, 제약조건 | 최적 버티포트 배치 | Optimizer Agent |
| `measure_before` | 네트워크 (대중교통만) | 접근성 지표 (전) | Evaluator Agent |
| `measure_after` | 네트워크 (통합) | 접근성 지표 (후) | Evaluator Agent |
| `compare_scenarios` | 시나리오 파라미터 | 전/후 비교 결과 | Evaluator Agent |
| `calc_equity` | 접근성 분포 | Gini 계수, Theil Index | Evaluator Agent |

## 5. UAM 도입 전/후 효과 평가 설계

### 5.1 평가 프레임워크

```
              ┌──────────────────┐
              │ 현재 네트워크     │
              │ (GTFS 대중교통)  │
              └────────┬─────────┘
                       │
          ┌────────────┼────────────┐
          ↓                         ↓
  ┌───────────────┐        ┌───────────────┐
  │  도입 전 측정  │        │  도입 후 측정  │
  │  - 접근성     │        │  - 접근성     │
  │  - 통행시간   │        │  + UAM 링크   │
  │  - 형평성     │        │  - 통행시간   │
  └───────┬───────┘        │  - 형평성     │
          │                └───────┬───────┘
          └────────────┬───────────┘
                       ↓
              ┌──────────────────┐
              │   비교 분석       │
              │  - 시간 절감     │
              │  - 접근성 변화   │
              │  - 형평성 변화   │
              │  - 수혜 지역 분포│
              └──────────────────┘
```

### 5.2 평가 지표

| 지표 | 설명 | 산출 방법 |
|------|------|-----------|
| 통행시간 단축 | OD별 시간 절감량 | (전 소요시간) - (후 소요시간) |
| 접근성 (Accessibility) | N분 내 도달 가능 기회 수 | 등시선 내 일자리/시설 수 |
| 형평성 (Equity) | 접근성 분포 균등도 | Gini 계수 (0=완전 균등, 1=완전 불균등) |
| 커버리지 | 버티포트 도보권 인구 비율 | 800m 반경 내 인구 / 전체 인구 |
| 비용효율성 | 시간 절감 대비 비용 | 절감 분당 추가 비용 |

### 5.3 시나리오 설계

| 시나리오 | 버티포트 수 | 요금 수준 | 운행 간격 | 분석 목적 |
|----------|:-----------:|:---------:|:---------:|-----------|
| S1 (최소) | 5개 | 고가 (택시 수준) | 15분 | 최소 투자 시 효과 |
| S2 (기본) | 10개 | 중가 | 10분 | 현실적 도입 시나리오 |
| S3 (확장) | 20개 | 저가 (버스 수준) | 5분 | 최대 확장 시 효과 |

## 8. 연구적 기여 (논문 관점)

### 8.1 Research Questions

> **RQ1.** 실제 스마트카드 OD 데이터 기반 UAM 수요 추정은 기존 방법론 대비 어떤 차이가 있는가?

> **RQ2.** UAM-대중교통 통합 네트워크에서 버티포트 최적 배치가 접근성과 형평성에 미치는 영향은?

> **RQ3.** UAM 도입이 대중교통 사각지대 해소에 실질적으로 기여하는가?

### 8.2 Contribution

1. **데이터 기반 수요 추정**: 실제 스마트카드 OD → UAM 전환 수요 (기존: 설문/인구밀도)
2. **멀티모달 네트워크 모델링**: GTFS + UAM 통합 네트워크에서 접근성 분석
3. **최적화 + 형평성**: 수요 최대화뿐 아니라 형평성 제약 고려
4. **전/후 효과 평가 프레임워크**: 다차원 지표(시간, 접근성, 형평성)로 체계적 비교
