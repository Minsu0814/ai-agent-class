# UAM 최적 버티포트 입지 분석 및 멀티모달 경로 탐색 시스템

## 1. 프로젝트 개요

현재 서울시 대중교통 데이터(스마트카드, GTFS)를 분석하여 **대중교통으로 불편한 구간**을 식별하고, **UAM(도심항공교통) 도입 시 통행 효율 개선 효과**를 분석하는 AI 기반 웹 서비스를 개발한다.

### 핵심 질문
> "서울시에서 UAM이 가장 필요한 구간은 어디이며, 도입 시 얼마나 효과적인가?"

---

## 2. 연구 배경 & 기존 연구의 갭 (Gap)

### 기존 UAM 버티포트 입지 연구의 한계

| 기존 연구 | 문제점 |
|-----------|--------|
| 설문조사 기반 수요 추정 | 실제 통행과 괴리 |
| 인구밀도/POI 기반 입지 선정 | 실제 OD 수요 미반영 |
| UAM만 단독 분석 | 기존 대중교통 네트워크와의 연계 부족 |
| 단순 시간 절감 분석 | 접근성, 형평성 등 다차원 분석 부족 |

### 본 연구의 차별점
- **실제 스마트카드 OD 데이터** 기반 수요 추정 (설문/인구밀도 X)
- **GTFS 네트워크 + UAM 통합** 멀티모달 분석
- **접근성 + 형평성** 다차원 효과 평가
- **UAM 도입 전/후 비교 분석** 프레임워크 제시

---

## 3. 연구 질문 (Research Questions)

> **RQ1.** 실제 대중교통 통행 데이터(스마트카드) 기반으로 UAM 수요를 추정하면, 기존 방법론(인구밀도, 설문) 대비 어떤 차이가 있는가?

> **RQ2.** UAM-대중교통 통합 네트워크에서 버티포트 최적 배치는 도시 교통 접근성(accessibility)과 형평성(equity)에 어떤 영향을 미치는가?

> **RQ3.** UAM 도입이 대중교통 사각지대 해소에 실질적으로 기여할 수 있는가?

## 4. 연구 Contribution

```
┌──────────────────────────────────────────────────┐
│  Contribution 1: 데이터 기반 수요 추정            │
│  - 스마트카드 OD → "UAM 전환 가능 통행" 추출      │
│  - 조건: 직선거리 대비 통행시간 비율 높은 OD       │
│  - 기존 설문/인구밀도 방식과 비교 검증             │
├──────────────────────────────────────────────────┤
│  Contribution 2: 멀티모달 네트워크 모델링          │
│  - GTFS 기반 대중교통 네트워크 구축                │
│  - UAM 가상 링크 추가 (버티포트 간)                │
│  - 통합 네트워크에서 접근성 지표 산출              │
├──────────────────────────────────────────────────┤
│  Contribution 3: 최적화 + 형평성 분석              │
│  - 버티포트 입지 최적화 (수요 최대화)              │
│  - 형평성 관점 분석 (소득/지역별 혜택 분배)        │
│  - → 부유한 지역만 혜택? 교통 약자 지역은?        │
├──────────────────────────────────────────────────┤
│  Contribution 4: UAM 도입 전/후 효과 평가          │
│  - 도입 전: GTFS 대중교통 네트워크만의 접근성      │
│  - 도입 후: UAM 통합 네트워크의 접근성             │
│  - 비교 지표: 통행시간, 접근성, 형평성(Gini 등)    │
│  - 시나리오별 민감도 분석                          │
└──────────────────────────────────────────────────┘
```

## 5. 방법론 프레임워크

```
Phase 1. 데이터 전처리 & OD 분석
  ├─ 스마트카드 → OD 행렬 구축
  ├─ GTFS → 대중교통 네트워크 그래프
  └─ 통행시간 vs 직선거리 비율 산출

Phase 2. UAM 전환 수요 추정
  ├─ 필터링: 장거리 + 고시간 + 고수요 OD
  ├─ UAM 경쟁력 지표 정의
  └─ 수요 추정 모델 (로짓 모델 등)

Phase 3. 버티포트 입지 최적화
  ├─ 후보지 선정 (클러스터링)
  ├─ 최적화 모델 (MILP or 메타휴리스틱)
  └─ 목적함수: 접근성 최대화 + 형평성

Phase 4. UAM 도입 전/후 효과 평가
  ├─ [전] GTFS 기반 대중교통 전용 네트워크 → 접근성 측정
  ├─ [후] 버티포트 + UAM 링크 추가 통합 네트워크 → 접근성 재측정
  ├─ 비교 분석
  │    ├─ 통행시간 단축량 (OD별, 지역별)
  │    ├─ 접근성 변화 (등시선도, 도달가능 범위)
  │    ├─ 형평성 변화 (Gini 계수, Theil Index)
  │    └─ 수혜 지역 vs 비수혜 지역 공간 분포
  └─ 시나리오 분석
       ├─ 버티포트 5개 / 10개 / 20개
       ├─ UAM 요금 수준별 (저가 / 중가 / 고가)
       └─ 운행 간격별 (5분 / 10분 / 15분)
```

## 7. 주요 기능

| 기능 | 사용 데이터 | 설명 |
|------|------------|------|
| 대중교통 불편 구간 히트맵 | 스마트카드 + GTFS | 통행시간 대비 직선거리 비율이 높은 OD 쌍 시각화 |
| 버티포트 입지 추천 | 스마트카드 (수요) | 클러스터링 기반 수요 밀집 지점 → 후보지 도출 |
| UAM 도입 전/후 비교 대시보드 | GTFS + UAM 가상노선 | 접근성·통행시간·형평성 변화 시각화 |
| 시나리오 시뮬레이션 | 전체 | 버티포트 수, 요금, 운행간격별 효과 비교 |
| AI 질의응답 | LLM API | 자연어로 구간 분석 및 효과 예측 질의 |

## 9. 기술 스택

```
Frontend  : Streamlit + Folium / Pydeck (지도 시각화)
Backend   : Python + Pandas + NetworkX (경로 분석)
최적화     : PuLP / Gurobi (MILP) 또는 메타휴리스틱
통계/ML   : Scikit-learn (클러스터링), Statsmodels (로짓)
AI        : Claude or GPT API (질의응답 + 데이터 분석)
Data      : 스마트카드 CSV + GTFS + UAM 가상노선
```

## 10. UAM 도입 전/후 효과 평가 상세

### 10.1 평가 지표

| 지표 | 설명 | 비교 방법 |
|------|------|-----------|
| 통행시간 | OD별 최단 통행시간 | 전/후 차이 (분) |
| 접근성 (Accessibility) | 일정 시간 내 도달 가능한 기회(일자리, 시설) 수 | 등시선도 비교 |
| 형평성 (Equity) | 접근성의 지역/소득별 분포 균등도 | Gini 계수, Theil Index |
| 커버리지 | 버티포트 도보권(800m) 내 인구 비율 | 인구 커버리지 % |
| 비용효율성 | 시간 절감 / 추가 비용 | 분당 비용 절감 효과 |

### 10.2 시나리오 설계

| 시나리오 | 버티포트 수 | 요금 수준 | 운행 간격 |
|----------|:-----------:|:---------:|:---------:|
| S1 (최소) | 5개 | 고가 (택시 수준) | 15분 |
| S2 (기본) | 10개 | 중가 | 10분 |
| S3 (확장) | 20개 | 저가 (버스 수준) | 5분 |

### 10.3 기대 결과물

- 서울시 전역 접근성 변화 지도 (전/후 비교)
- OD별 통행시간 단축 Top-K 구간
- 형평성 지표 변화 차트
- 시나리오별 민감도 분석 결과

## 12. 기대 효과

- **시의성**: 서울시 K-UAM 실증 사업과 연계 가능한 분석 결과 제공
- **데이터 기반 의사결정**: 실제 통행 데이터 기반 버티포트 입지 제안
- **정책 기여**: UAM 도입의 형평성 영향 분석 → 정책 시사점 도출
- **Agentic AI 활용**: LLM이 분석 도구를 직접 호출하여 자연어 질의에 응답
- **시각적 결과물**: 지도 기반 대시보드로 직관적 데모 가능
